# Learner Guide - Certified Lean Six Sigma Green Belt (CLSSGB) Training

## Course Overview

This learner guide supports a practical Certified Lean Six Sigma Green Belt course. It is designed for learners who facilitate improvement events, map work processes, analyze basic performance data, solve operational problems, and implement countermeasures under the DMAIC method.

The course follows the Green Belt study domains:

1. Basics and background of Lean Six Sigma.
2. Process mapping.
3. Facilitation.
4. Analyze data.
5. Problem solving.
6. Lean Six Sigma countermeasures.

## Before You Start

### Recommended Project Scenario

Use one realistic process throughout the course. Examples:

- Course registration form errors.
- Late invoice approvals.
- Delayed customer support responses.
- Rework in document submission.
- Long onboarding cycle time.

## Learning Outcomes

By the end of the course, you should be able to:

1. Explain Lean, Six Sigma, DMAIC, COPQ, and the Green Belt role.
2. Select a suitable improvement project.
3. Build SIPOC and process maps.
4. Identify value-added and non-value-added work.
5. Facilitate a small Kaizen event.
6. Create an A3 report.
7. Define KPIs and collect baseline data.
8. Build run charts and Pareto charts.
9. Calculate basic performance metrics such as yield, DPU, and DPMO.

## Lab 01 Guide - Lean Six Sigma Basics and Project Selection

### Objectives

- Explain Lean Six Sigma basics.
- Identify Green Belt responsibilities.
- Link business problems to customer value.
- Select a suitable project.

### Steps

1. Write a short definition of Lean, Six Sigma, and DMAIC in your own words.
2. List what a Green Belt can lead independently and where a Black Belt or sponsor may be needed.
3. Identify one process problem from your workplace or a trainer scenario.
4. Estimate the cost of poor quality, including rework, delay, customer dissatisfaction, and employee frustration.
5. Identify customers and what they value.
6. Use the Kano model to classify basic needs, performance needs, and delight factors.
7. Write the transfer function `Y=f(x)` for your project: define the output Y and likely input X factors.
8. Score the project for impact, feasibility, data availability, and scope.
9. Select one project for the course.
10. Write a draft problem statement and goal statement.

### Deliverables

- Project selection worksheet.
- Draft problem statement.
- Draft goal statement.
- COPQ notes.
- Kano notes.

### Checkpoint

You can explain why this project is worth doing and how it fits the Green Belt role.

## Lab 02 Guide - SIPOC, Process Map, and Eight Wastes

### Objectives

- Scope the process with SIPOC.
- Build a current-state process map.
- Identify value-added and non-value-added work.
- Locate the eight wastes.

### Steps

1. Define the start and end points of the process.
2. List suppliers, inputs, process steps, outputs, and customers.
3. Create a SIPOC table.
4. Draw a process flowchart with major steps and decision points.
5. Mark handoffs, queues, rework loops, approvals, and waiting points.
6. Label each step as value-added, business-value-added, or non-value-added.
7. Identify examples of transportation, inventory, motion, waiting, overproduction, overprocessing, defects, and underused skills.
8. Choose two waste areas that may be suitable for improvement.
9. Update the problem statement if the map changes your understanding of scope.

### Deliverables

- SIPOC.
- Current-state process map.
- Waste analysis table.
- Updated scope notes.

### Checkpoint

You can show where waste occurs in the process and explain which areas deserve measurement.

## Lab 03 Guide - Facilitation, Stakeholders, Kaizen, and A3

### Objectives

- Analyze stakeholders.
- Plan a small Kaizen event.
- Practise facilitation behaviors.
- Start an A3 report.

### Steps

1. List people affected by the process problem.
2. Create a stakeholder grid using influence and interest.
3. Identify possible supporters, blockers, and subject matter experts.
4. Write a facilitation plan for a 60-minute improvement meeting.
5. Include agenda, objective, ground rules, roles, and expected outputs.
6. Plan a Kaizen event around one process pain point.
7. Define pre-work, event activities, and follow-up actions.
8. Create an A3 report with background, current condition, target condition, and next steps.
9. Practise explaining the A3 in three minutes.
10. Capture feedback and revise the A3.

### Deliverables

- Stakeholder analysis.
- Facilitation agenda.
- Kaizen plan.
- Draft A3 report.

### Checkpoint

You can facilitate a structured improvement conversation without jumping straight to solutions.

## Lab 04 Guide - Data Collection, KPIs, Run Charts, and Pareto

### Objectives

- Define KPIs and operational definitions.
- Create a data collection plan.
- Build a run chart.
- Build a Pareto chart.

### Steps

1. Select one primary KPI for the project.
2. Write an operational definition for the KPI.
3. Define unit, defect, opportunity, target, and data source if applicable.
4. Create a data collection plan with owner, frequency, sample size, and method.
5. Collect or use sample baseline data.
6. Check the data for missing values and inconsistent categories.
7. Create a run chart showing performance over time.
8. Mark known events or process changes on the chart.
9. Create a Pareto chart for defect categories or delay reasons.
10. Identify the largest contributors to the problem.
11. Decide where root cause analysis should focus.

### Deliverables

- KPI definition.
- Data collection plan.
- Clean baseline data table.
- Run chart.
- Pareto chart.

### Checkpoint

You can explain the process baseline using data rather than opinion.

## Lab 05 Guide - Basic Statistics and Process Capability

### Objectives

- Summarize data with basic statistics.
- Understand variation.
- Calculate common Six Sigma metrics.
- Interpret basic process capability.

### Steps

1. Import the baseline data from Lab 04.
2. Calculate count, mean, median, minimum, maximum, range, and standard deviation.
3. Create a histogram if the data is continuous.
4. Calculate yield if good and total units are known.
5. Calculate DPU if defects and units are known.
6. Calculate DPMO if opportunities are known.
7. Estimate sigma level using a trainer-provided conversion table if available.
8. If specification limits are known, compare actual performance against target.
9. Discuss whether the process is predictable enough for simple capability interpretation.
10. Write a short baseline performance summary.

### Deliverables

- Basic statistics table.
- Histogram if applicable.
- Yield, DPU, DPMO, or sigma worksheet.
- Baseline summary.

### Checkpoint

You can describe process performance with center, spread, and defect metrics.

## Lab 06 Guide - Root Cause Analysis and FMEA

### Objectives

- Identify possible root causes.
- Use 5 Whys and Fishbone analysis.
- Prioritize root causes.
- Apply FMEA to process risk.

### Steps

1. Choose the top issue from the Pareto chart.
2. Create a fishbone diagram using categories such as people, process, materials, equipment, measurement, environment, and management.
3. Select two suspected causes and complete 5 Whys.
4. Identify which causes need evidence before action.
5. Check available data or collect quick observations to support or reject causes.
6. Create an FMEA for the process step most affected by the problem.
7. Identify failure mode, effect, cause, and current control.
8. Score severity, occurrence, and detection.
9. Calculate RPN.
10. Prioritize risks for countermeasures.

### Deliverables

- Fishbone diagram.
- 5 Whys worksheet.
- Evidence notes.
- FMEA.
- Prioritized root cause list.

### Checkpoint

You can separate suspected causes from evidence-supported causes and identify the highest process risks.

## Lab 07 Guide - Countermeasures, 5S, Kanban, and Poka-Yoke

### Objectives

- Select countermeasures linked to root causes.
- Apply 5S and visual management.
- Use Kanban concepts.
- Design mistake-proofing and standard work.

### Steps

1. List validated root causes and high-priority FMEA risks.
2. Brainstorm possible countermeasures.
3. Remove countermeasures that do not address a real cause.
4. Use an impact-effort matrix to prioritize options.
5. Apply 5S to the selected process area or information workspace.
6. Identify where a Kanban signal could prevent shortage, overload, or waiting.
7. Design a poka-yoke or error-proofing idea.
8. Draft standard work for the improved process step.
9. Define how the solution will be tested on a small scale.
10. Record expected benefits and possible side effects.

### Deliverables

- Countermeasure list.
- Impact-effort matrix.
- 5S action list.
- Kanban or visual control sketch.
- Poka-yoke idea.
- Draft standard work.

### Checkpoint

You can explain how each countermeasure addresses a root cause or risk.

## Lab 08 Guide - Control Plan, Standard Work, and Project Close

### Objectives

- Create a control plan.
- Define response actions.
- Prepare training and handover.
- Complete the A3 report.

### Steps

1. List the improved process steps that must be sustained.
2. Define the KPI or CTQ to monitor.
3. Define target, specification, measurement method, frequency, and owner.
4. Write a response plan for performance outside the expected range.
5. Finalize standard work instructions.
6. Plan how team members will be trained.
7. Define audit or check-in frequency.
8. Compare before and after performance if pilot data is available.
9. Update the A3 report with root cause, countermeasure, result, and follow-up.
10. Prepare a five-minute project presentation.
11. Present the project story to a peer or trainer.
12. Record lessons learned.

### Deliverables

- Control plan.
- Response plan.
- Final standard work.
- Final A3 report.
- Project presentation notes.

### Checkpoint

You can show how the process owner will sustain the improvement after the project closes.
